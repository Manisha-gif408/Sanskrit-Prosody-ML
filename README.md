# Sanskrit-Prosody-ML
The system employs a custom Syllabic Weight Engine to parse Devanagari text into Laghu and Guru patterns based on Pingala’s Chanda-śāstra. Using Digital Signal Processing (DSP) via the Librosa library, the system applies time-stretching and pitch-contouring to align the audio output with specific metrical frameworks .
Technical Innovation: The 'Deep-Vedic' EngineUnlike standard TTS which is monotone, our engine uses:Prosody Mapping: Assigning $2\times$ duration to Guru (long) syllables and $1\times$ to Laghu (short) syllables.Acoustic Convolution: We don't use simple "echo." We use Signal Convolution with a stone-temple impulse response to simulate the $Garbhagriha$ (Sanctum) acoustics.Whisper-Validation: A closed-loop system where the AI "listens" to its own generated priest-voice to ensure 95%+ phonetic accuracy.
